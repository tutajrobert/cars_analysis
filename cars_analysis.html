Analiza ogłoszeń sprzedaży samochodów używanych¶

0. Zbiór danych¶

Analiza dotyczy zbioru danych udostpnionego w portalu kaggle.com, w ktrym znajduj si dane sprzedawanych samochodw uywanych pobrane z niemieckiego portalu aukcyjnego ebay. Oryginalne dane w pliku csv dostpne s pod niniejszym adresem https://www.kaggle.com/orgesleka/used-cars-database

0.1. Cel projektu¶

Cele projektu s nastpujce:

  • przedstawienie w praktyce umiejtnoci i wiedzy nabytych w trakcie studiw podyplomowych
  • przyswojenie i spenienie wymaga stawianych przez schemat postpowania CRISP-DM
  • utworzenie modelu regresji liniowej do szacowania ceny samochodu uywanego

0.2. Wymagania projektu¶

Wymagania co do projektu s nastpujce:

  • spenienie celw z Podrozdziau 0.1.
  • wykonanie raportu obrazujcego przebieg postpowania analizy danych

1.2. Przygotowanie danych kategorycznych¶

Dla kolumny seller wystpuj dwie kategorie, z ktrych jedna zawiera jedynie 3 elementy. Z tego powodu ta kolumna zostanie usunita w dalszym etapie, poniewa nie wnosi adnych danych do zbioru. Podobnie jest w przypadku kolumny offerType. Tutaj kategoria Gesuch zawiera tylko 12 elementw, a wic i ta kolumna zostanie usunita. Wykluczona ze zbioru danych zostanie rwnie kolumna abtest, poniewa danych w niej nie mona zinterpretowa.

Kolumna gearbox jest istotna, a w dodatku nie zawiera pustych elementw. Zostanie wic pozostawiona w niezmienionej formie.

W kolumnie fuelType jest 7 kategorii, lecz niektre z nich zawieraj ma liczb obserwacji (poniej 600). Tak jest w przypadku kategorii andere, elektro, hybrid oraz cng. Ze wzgldu na podobiestwo samochodw z instalacjami gazowymi zostanie utworzona wsplna kategoria gas dla otrzymanych kategorii lpg i cng. Pozostae z wymienionych kategorii zostan usunite wraz z danymi wewntrz nich.

Kolumna notRepairedDamage zawiera bardzo wane informacje o tym czy samochd jest uszkodzony czy naprawiony lub nieuszkodzony. Dane te wydaj si istotne dla dalszej analizy, mimo tego, e zawieraj a 72060 pustych obserwacji. Puste elementy zostan usunite.

Kolumna brand zawiera marki wystawionych na sprzeda samochodw. Jest w niej 39 kategorii, z ktrych najwicej, 79640 obserwacji zawiera kategoria volkswagen. Ta liczba stanowi 21% wszystkich danych.

W kolumnie model wystpuje 251 kategorii i dane w niej s mocno rozproszone. Najczstsz obserwacj w tej kolumnie jest golf w liczbie 30070, co stanowi 8% caego zbioru danych. Niestety, ta kolumna zostanie usunita ze zbioru. Wykorzystanie jej na cele analizy wymagaoby uycia bardziej zaawansowanych metod ni przewidziano na cel niniejszego zadania z powodu duego rozproszenia liczby obserwacji w katgoriach. Zostao to przedstawione na Wykresie 1.

Kolumna vehicleType ma 7 kategorii, z czego jedna z nich dotyczy samochodw opisanch jako inne - andere. Dalej przedstawiono wycinek nazw ogosze samochodw z tej kategorii. Jak wida zawiera ona samochody cakowicie zniszczone, do naprawy lub przerabiane. Takie egzemplarze nie wpisuj si w og analizowanych danych i zostan usunite.

#definicja funkcji dla okrelenia kryterium BIC oraz AIC
def BIC(mse, p, n):
    return n*np.log(mse) + p*np.log(n)

def AIC(mse, p, n):
    return n*np.log(mse) + 2*p

#definicja funkcji przeprowadzajcej regresj liniow i podsumowujcej parametry
def fit_regression(X_ucz, X_test, y_ucz, y_test):
    r = sklearn.linear_model.LinearRegression()
    r.fit(X_ucz, y_ucz)
    y_ucz_pred = r.predict(X_ucz)
    y_test_pred = r.predict(X_test)
    mse = sklearn.metrics.mean_squared_error
    mae = sklearn.metrics.mean_absolute_error
    n = X_ucz.shape[0]
    p = X_ucz.shape[1]
    return {
        "r_score" : r.score(X_ucz, y_ucz),
        "MSE_u" : mse(y_ucz, y_ucz_pred),
        "MAE_u" : mae(y_ucz, y_ucz_pred),
        "MSE_t" : mse(y_test, y_test_pred),
        "MAE_t" : mae(y_test, y_test_pred),
        "BIC_u" : BIC(mse(y_ucz, y_ucz_pred), p + 1, n),
        "AIC_u" : AIC(mse(y_ucz, y_ucz_pred), p + 1, n),
        "zmienne" : p
    }

def forward_selection_BIC(X, y):
    n, m = X.shape
    best_idx = []
    best_free = set(range(m))
    best_fit = np.inf
    res = []
    for i in range(0, m):
        cur_idx = -1
        cur_fit = np.inf
        for e in best_free:
            r = sklearn.linear_model.LinearRegression()
            test_idx = best_idx + [e]
            r.fit(X[:, test_idx], y)
            test_fit = BIC(sklearn.metrics.mean_squared_error(y, r.predict(X[:,test_idx])), i+2, n)
            if test_fit < cur_fit: cur_idx, cur_fit = e, test_fit
        if cur_fit > best_fit: break
        best_idx, best_fit = best_idx + [cur_idx], cur_fit
        best_free.discard(cur_idx)
        res.append((cur_idx, cur_fit))
    return res

def forward_selection_AIC(X, y):
    n, m = X.shape
    best_idx = []
    best_free = set(range(m))
    best_fit = np.inf
    res = []
    for i in range(0, m):
        cur_idx = -1
        cur_fit = np.inf
        for e in best_free:
            r = sklearn.linear_model.LinearRegression()
            test_idx = best_idx + [e]
            r.fit(X[:, test_idx], y)
            test_fit = AIC(sklearn.metrics.mean_squared_error(y, r.predict(X[:,test_idx])), i+2, n)
            if test_fit < cur_fit: cur_idx, cur_fit = e, test_fit
        if cur_fit > best_fit: break
        best_idx, best_fit = best_idx + [cur_idx], cur_fit
        best_free.discard(cur_idx)
        res.append((cur_idx, cur_fit))
    return res

5. Wnioski¶

Niestety przez nie wystarczajc moc komputera (pami, procesor) nie udao si wykorzysta wszystkich danych do analizy. Mimo to, wyniki s porawne i mniej wicej przy pomocy utworzonych modeli daje si szacowa cen samochodu. Ta jednak zaley od wielu wicej czynnikw, o ktrych nie ma adnych informacji w danych. Dlatego moe nie jest zdziwieniem do dua rednia z moduu rnic dla modeli (okoo 2000 euro).

Dla modelu wielomianowego udao uzyska si wysoki wspczynnik R2 wynoszcy 0.83. Wybr tylko czci zmiennych w oparciu o kryterium BIC lub AIC nie przynioso poprawy miary dopasowania i bdw, lecz odchudzio model ze zmiennych, co take jest wane.

Co można poprawić¶

  1. Podczas przygotowania danych usunitych zostao wiele rekordw pustych dla kolumn, ktre jak si okazao pniej nie s istotne na cen np. rodzaj skrzyni biegw. W ponownej analizie warto rozway pozostawienie tych rekordw co zwikszy liczb danych w analizie regresji.

  2. By poprawi wyniki redniego moduu rnic mona rozway analiz na wycinku danych tylko dla danego typu samochodw lub w danym zakresie cenowym. Wydaje si, e rozrzut obserwacji w zbiorze jest bardzo duy nawet po przygotowaniu co jest nie bez znaczenia na przewidywanie ceny.

  3. Zadanie zostao wykonane tylko dla jednego stosunku liczby obserwacji w zbiorze uczcym do testowego. Warto rozway sprawdzenie dla innych wartoci.

  4. W analizie zosta wykorzystny model regresji o domylnych ustawieniach. By moe przez zmian ustawie daoby si uzyka lepsze wartoci dopasowania czy mniejsze bdy.
